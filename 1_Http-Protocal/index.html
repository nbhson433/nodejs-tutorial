# HTTP Protocol

Cách thức website được truyền tải thông qua internet
Website được lưu trữ ở một cái máy chủ ở data center, một tòa nhà nào đó
(Địa chỉ máy chủ: Tablet network->Headers->General->Remote Address)
Dựa vào url thì máy chủ sẽ biết được cần trả về cái gì (ctrl + U)
Source code được trả về và trình duyệt sẽ đọc các mã nguồn đó để render và hiển thị

# Request Method

GET Được sử dụng để lấy lại thông tin từ Server đã cung cấp bởi sử dụng một URI đã cung cấp.
Các yêu cầu sử dụng GET chỉ nhận dữ liệu và không có ảnh hưởng gì tới dữ liệu.

HEAD Tương tự như GET, nhưng nó truyền tải dòng trạng thái và khu vực Header.

POST Một yêu cầu POST được sử dụng để gửi dữ liệu tới Server
Ví dụ, thông tin khách hàng, file tải lên,...

PUT Thay đổi tất cả các đại diện hiện tại của nguồn mục tiêu với nội dung được tải lên.

DELETE Gỡ bỏ tất cả các đại diện hiện tại của nguồn mục tiêu bởi URI.

CONNECT Thiết lập một tunnel tới Server được xác định bởi URI đã cung cấp.

OPTIONS Miêu tả các chức năng giao tiếp cho nguồn mục tiêu.

TRACE Trình bày một vòng lặp kiểm tra thông báo song song với path tới nguồn mục tiêu.

# HTTP Status Code

1xx: Thông tin
2xx: Thành Công
3xx: Sự điều hướng lại
4xx: Lỗi Client
5xx: Lỗi Server
